Assignment 2

DESCRIPTION

This assignment involves creating an interactive for an audience of our choice. We are tasked with utilizing several form controls that are common among event-driven applications. During this phase of learning Javascript, we are simply interacting with embedded functions that serve to work on the fly without using a backend database or a built-in API library of javascript functions.

REFLECTION

I've learned that the <head> tag is utilized for embedding javascript (js) functions, which are run by the CPU during the loading phase. I also learned that js is an interpreted language, meaning most data types are predefined and don't have to be compiled before being executed. The emphasis on placement of js is not restricted to the body, however, the importance of knowing why certain js code is placed in the <head> tag versus the <body> tag is crucial.

Forms are easy to work with, but form design takes some getting used to. There are several business and personal uses for form interaction and I found this to be the most difficult. Once I got an idea of what to use, the coding went quickly because js follows the same basic programming principles as most Object Oriented languages like Java and C# with the added advantage/disadvantage that it doesn't need to be compiled.

LEARNING OBJECTIVES

  • Design an Interactive Form using embeded Javascript
  • Utilize Event-Driven form controls
  1. Radial Button
  2. Checkbox
  3. Drop-Down List
  4. Input Boxes
  • Create interactive functions
  • Use if/then/else | Switch/Case conditions
  • Use for/while loops
  • Validate input parameters

Assignment 3

DESCRIPTION

Assignment 3 introduces the use of an external javascript file. In this case I created a library of basic statistical calculations. Simply enter up to 9 numbers and select the Statistics button to get a summary of the resultant calculations.

REFLECTION

instead of using the <head> tag for embedding javascript (js) functions, we could easily link to an external js file, which is beneficial when you have a large website with multiple interactions. It helps for readability of code and allows each method to be loaded when it's called rather than each time the landing page is loaded.
